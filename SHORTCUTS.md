# Kele 快捷键参考

**版本**: v0.1.3
**更新时间**: 2026-02-06

---

## ⌨️ 全局快捷键

### 基础操作

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Enter` | 发送消息 | 发送输入的消息给 AI |
| `Tab` | 命令补全 | 自动补全斜杠命令 |
| `ESC` | 中断任务 | 停止当前流式响应 |
| `Ctrl+C` | 退出程序 | 完全退出 Kele |

---

## 📝 详细说明

### Enter - 发送消息

**功能**: 发送当前输入框的内容

**行为**:
- 普通文本 → 发送给 AI 处理
- 斜杠命令 → 执行相应命令
- 空内容 → 忽略

**示例**:
```
你好<Enter>        → 发送消息给 AI
/help<Enter>       → 执行帮助命令
```

---

### Tab - 命令补全

**功能**: 智能补全斜杠命令

**行为**:
- 唯一匹配 → 直接补全并添加空格
- 多个匹配 → 显示候选列表 + 补全公共前缀
- 无匹配 → 保持原样

**示例**:
```
/mo<Tab>           → /model
/mod<Tab>          → 显示候选: /model, /models, /model-reset
/model-<Tab>       → /model-reset
```

**提示**:
- 只对以 `/` 开头的输入生效
- 不区分大小写
- 保留已输入的参数

---

### ESC - 中断任务

**功能**: 中断当前正在进行的任务

**行为**:
- 流式响应中 → 停止接收内容，标记为已中断
- 无任务运行 → 显示提示信息

**示例**:
```
# AI 正在长篇回复时
<ESC>              → 停止接收，可以立即输入新消息

# 没有任务运行时
<ESC>              → 显示: "💡 使用 /exit 或 Ctrl+C 退出程序"
```

**用途**:
- 停止冗长的回复
- 取消错误的请求
- 快速切换话题
- 节省 API 用量

**注意**:
- ESC **不会**退出程序，只中断当前任务
- 中断后可以立即开始新对话
- 支持"多任务"工作流

---

### Ctrl+C - 退出程序

**功能**: 完全退出 Kele 程序

**行为**:
- 立即终止程序
- 不保存当前输入
- 自动保存会话历史

**替代方法**:
- `/exit` 命令
- `/quit` 命令

## 🎯 使用场景

### 场景 1: 快速切换话题

```
用户: 帮我写一篇长文章
AI: 正在写作中...  ← 按 ESC 中断
用户: 不用了，帮我写首诗吧
```

### 场景 2: 纠正错误请求

```
用户: 删除所有文件  ← 刚发送就后悔了
    ↓ 立即按 ESC
AI: [已中断]
用户: 列出所有文件
```

### 场景 3: 节省成本

```
AI: 正在生成超长代码...  ← 已经够了
    ↓ 按 ESC 停止
AI: [已中断]
用户: 就这么多够了，谢谢
```

### 场景 4: 高效工作流

```
# 快速尝试多个模型
/model gpt-4o<Enter>
> 测试问题<Enter>
AI: 开始回复...  ← ESC 中断
/model claude-3-5-sonnet<Enter>
> 测试问题<Enter>
```

## 💡 快捷键组合

### 命令快速输入

```
/st<Tab><Enter>    → 执行 /status 命令
/cl<Tab><Enter>    → 执行 /clear 命令
/mo<Tab>gpt<Enter> → 执行 /model gpt-4o
```

### 工作流优化

```
1. /clear<Enter>   # 清空历史
2. /model ...<Tab><Enter>  # 切换模型
3. 开始新任务...
4. <ESC> 如需中断
5. <Ctrl+C> 完成工作后退出
```

## 🚫 常见误区

### ❌ 错误理解

**误区 1**: ESC 退出程序
- ❌ 错误: ESC 会关闭 Kele
- ✅ 正确: ESC 只中断任务，不退出程序

**误区 2**: 无法中断流式响应
- ❌ 错误: 必须等 AI 回复完
- ✅ 正确: 随时按 ESC 中断

**误区 3**: Tab 只能补全命令名
- ❌ 错误: Tab 后必须手动输入参数
- ✅ 正确: Tab 保留已有参数

### ✅ 正确使用

```
# 场景: 需要重新开始
输入错了 → <ESC> → 清空输入 → 重新输入

# 场景: AI 回复太长
AI 回复中 → <ESC> → 中断 → 继续新对话

# 场景: 快速输入命令
/sta<Tab><Enter> → 立即执行 /status
```
